PROTOCOL

How it works

GitBlock runs on one principle: AI inference should be free, decentralized, and owned by the community. No central server. No API keys. Just open models running on distributed nodes.

1

Node operators register

Anyone with a GPU can run a GitBlock node. Install the software, register on-chain, and start serving open-source models like Llama, Mistral, and Gemma.

2

Users send prompts

Users send questions through the web app, Telegram bot, or API. The network routes the request to the best available node based on speed, reliability, and model availability.

3

Inference happens on-chain

Every inference request is recorded on-chain. The node serves the response, the quality is scored, and the interaction is permanently logged for transparency.

4

Tokens reward contributors

Node operators earn $GBLOCK tokens for serving quality inference. Reputation scores determine traffic distribution. Good nodes earn more. Bad nodes get deprioritized.

SECURITY

Trustless by design. Cryptographically verifiable.

Every inference is on-chain verifiable. Node reputation is immutable. Quality is provable.

01

On-chain proof of inference

Every inference request and response is hashed and recorded on-chain. Users can verify their AI responses were served by legitimate nodes.

02

Stake-slashed quality

Nodes stake $GBLOCK tokens as collateral. Serving bad responses, going offline, or manipulating results triggers automatic slashing.

03

Zero-knowledge verification

ZK proofs verify model quality without exposing user prompts. Privacy-preserving quality assurance at the protocol level.

04

Decrypted end-to-end

Prompts are encrypted client-side, decrypted only on the serving node. No central server ever sees your data. Not even us.
